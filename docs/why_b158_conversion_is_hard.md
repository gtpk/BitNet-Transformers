## The Better Question

The original hope was:

```text
Can we convert an existing FP LLM to BitNet b1.58 the way we quantize a model?
```

The better research question is now:

```text
BitNet b1.58 works when trained natively. Why is existing-model conversion to
b1.58 much harder than ordinary quantization?
```

This reframes the project from "make it work at all costs" to "identify the exact
barriers between standard quantization and native ternary training."

## Short Answer

Ordinary quantization is usually a **small perturbation** of a full-precision model.
b1.58 ternarization is not. It is a move from a high-resolution real-valued parameter
space to a very small integer lattice:

```text
W_fp in R^{out x in}
W_b1.58 = gamma * T
T in {-1, 0, +1}^{out x in}
```

The native BitNet recipe can train the whole network so that this lattice is the
model's natural parameter space. Existing-model conversion starts from a model whose
representations, scales, and layer-to-layer activation distributions were learned in
continuous high precision. Forcing it onto the ternary lattice creates a global
distribution shift, not just local weight noise.

## Barrier 1: The Error Is Too Large To Be "Just Quantization"

For normal low-bit quantization, the starting assumption is:

```text
W_q = W + small error
```

For b1.58, the approximation is:

```text
W_q = gamma * T
T in {-1,0,+1}
```

The space of possible weights collapses drastically. Even if the runtime stores I2_S
faithfully, the converted model may be far from the FP function.

Our evidence:

- RT-114/115: one-shot PTQ has terrible absolute PPL at 160M/1.1B.
- RT-121: 160M one-shot ternary PPL `135,309` while Q2_K one-shot PPL is `97.9`.
- RT-120: 1.1B teacher-free recovery improves PPL to `162.5`, but FP is still `10.1`.

The useful metric is not only recovered fraction:

```text
residual_gap = CE_adapted - CE_fp = log(PPL_adapted / PPL_fp)
```

RT-120:

```text
log(162.5 / 10.1) ~= 2.78 nats
```

That residual gap is large enough that usable greedy generation should not be
expected.

## Barrier 2: Layer Errors Are Not Additive

A Transformer is a composition:

```text
h_L = F_L o F_{L-1} o ... o F_1(x)
```

Post-hoc mixed-bit repair assumes something like:

```text
total_gain(S) ~= sum_i gain_i
```

RT-123 challenges this. Restoring a single group to higher precision often made CE
worse, not better. The reason is that the all-ternary adapted model is a self-
consistent, though weak, fixed point. Replacing one local operator changes the
activation distribution seen by every downstream ternary operator:

```text
observed_gain_i = precision_benefit_i - mismatch_penalty_i
```

So a single-layer sensitivity scan is a biased estimator of true mixed-bit benefit.

Implication:

```text
The problem is not simply "choose the right layers."
It is "maintain a globally consistent activation distribution under a ternary lattice."
```

## Barrier 3: Teacher-Forced CE Is Not Free-Running Generation

CE/PPL evaluates:

```text
E_{x from data}[-log p(x_t | x_<t)]
```

Generation evaluates the model on its own contexts:

```text
x_t generated by the model itself
```

RT-122 showed this gap directly. The 1.1B adapted model has much better PPL than PTQ,
but greedy generation falls into WikiText-style section-header loops:

```text
"the 19th century . = = = = ="
```

This is not an I2_S runtime bug. Adapted I2_S and adapted F16 produce the same quality
tier and often the same degeneracy. The converted model distribution itself is
unstable under free-running decoding.

Implication:

```text
Recovery must target both teacher-forced loss and free-run stability.
```

## Barrier 4: Runtime Fidelity Is Solved, Quality Is Not

The project separated two questions:

```text
Can b1.58 weights run faithfully and efficiently?
Can an existing FP model be converted into good b1.58 weights?
```

The first question is mostly solved on x86:

- RT-111: official bitnet.cpp I2_S f32/i2_s parity on x86.
- RT-112: our Path A' b1.58 artifact runs with f16/f32 parity.
- RT-113/114/115: storage and token-gen speedups scale with model size.

The second question remains hard:

- RT-121: OURS does not beat Q2_K on PPL.
- RT-122: 1.1B all-I2_S adapted generation is not usable.
- RT-123: simple local mixed-bit repair violates the additive assumption.

Implication:

```text
The bottleneck is not I2_S export. The bottleneck is learning or reconstructing a
self-consistent b1.58-compatible function.
```

## Why Native BitNet Avoids This

Native BitNet-style training does not start with a high-precision solution and crush
it into a ternary lattice after the fact. It trains the representations while the
ternary constraint is part of the model's life.

In simplified terms:

```text
conversion: find ternary weights near an FP solution
native: find a good solution inside the ternary-constrained family
```

Those are different optimization problems. The second can shape internal
representations, activation scales, and layer interactions around ternary arithmetic
from the beginning.

This explains why short teacher-free CE helps but does not fully solve the problem:

```text
it moves the checkpoint toward a ternary-compatible basin,
but may not move far enough to rebuild stable free-running dynamics.
```

## What The Project Has Actually Learned

Positive:

```text
b1.58 I2_S is an excellent systems substrate when the weights are already compatible.
teacher-free CE can rescue catastrophic ternary PTQ by orders of magnitude.
runtime/export is faithful; I2_S preserves both improvements and degeneracy.
```

Negative:

```text
short teacher-free conversion does not yet beat Q2_K on quality.
1.1B all-I2_S adapted generation is not usable.
single-layer mixed-bit sensitivity is non-additive and can mislead DP selection.
```

Reframed contribution:

```text
This project maps the systems promise and conversion limits of b1.58 for existing
dense LLaMA checkpoints.
```

## What A Real Solution Probably Needs

A successful existing-model b1.58 conversion likely needs at least one of:

1. **activation-aware ternarization**

   Optimize the layer output, not only weight statistics:

   ```text
   min_{gamma,T} E_x ||W x - gamma T x||^2
   ```

2. **block input-output reconstruction**

   Match block functions on calibration activations:

   ```text
   F_l(h) ~= Q_l(h)
   ```

3. **hybrid-aware re-adaptation**

   Do not patch a converged all-I2_S model one layer at a time. Choose the topology
   first, then re-adapt under that topology.

4. **small residual correction**

   Use b1.58 as the fast core and add a constrained residual:

   ```text
   W ~= gamma T + A B
   ```

5. **free-run stability objective**

   Add data cleaning, repetition penalties during evaluation, or explicit loop-
   avoiding losses such as unlikelihood terms. CE alone can improve PPL while leaving
   greedy attractors intact.

## Decision

The current project should not claim:

```text
existing FP LLMs can be quantized to usable b1.58 like ordinary quantization
```

It can claim:

```text
existing-model b1.58 conversion is possible at the runtime/export level,
but quality conversion is a fundamentally harder problem than ordinary quantization
because it must rebuild a self-consistent ternary-lattice model.
```

The next experiments should be framed as attempts to close that gap, not as routine
quantization engineering.
